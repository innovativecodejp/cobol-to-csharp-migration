# 2026-02-18_Mvp14

## 🎯 今日の目的
MVP14として、INDEXEDファイルに対する `START ... KEY >= ...` と `READ ... NEXT` の最小検証を行う。

## 🛠 実施内容
- `samples/cobol/Mvp14Program.cbl` を新規作成
- `samples/data/mvp14/expected1.txt` / `expected2.txt` を追加
- `apps/backend/tests/CobolMvpRuntimeMvp14Tests.cs` を追加
- C#ランタイム側に `Mvp14Program` を追加し、`MVP14_INDEX` と `MVP14_CASE` による実行時制御を実装
- プログラム内で最小のインデックスデータ（0001/0003/0005）を生成後、開始キーでポジショニングして順次取得する流れを構成
- `dotnet test --filter CobolMvpRuntimeMvp14Tests` を実行してケース1/2を検証

## 🔍 結果
- Mvp14テストは 2件合格（失敗0）
- ケース1（STARTKEY=0003）で `0003`,`0005` の順で出力されることを確認
- ケース2（STARTKEY=0002）でも `KEY >=` の最初の一致が `0003` となり、同一結果になることを確認
- 最終行 `DONE` まで expected と完全一致

## 💡 学び
STARTの検証は、入力ファイル依存を避けてテスト内でデータ生成まで完結させると、
ポジショニング仕様そのものに焦点を当てやすい。

## 🧠 思考整理
MVP12（READ）とMVP13（WRITE）で基礎I/Oを分離したうえでMVP14（START）を追加したことで、
順次・出力・キー位置決めの責務を段階的に説明できる構成になった。
