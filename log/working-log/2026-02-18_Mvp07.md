# 202t-02-18_Mvp07

## 🎯 今日の目的
MVP07として、EVALUATEの最小検証（複数WHEN + THRU + OTHER）を追加し、分岐変換の信頼性を確保する。

## 🛠 実施内容
- `samples/cobol/Mvp07Program.cbl` を新規作成
- `samples/data/mvp07/input1.txt` / `expected1.txt` を追加
- `samples/data/mvp07/input2.txt` / `expected2.txt` を追加
- `apps/backend/tests/CobolMvpRuntimeMvp07Tests.cs` を追加
- C#ランタイム側に `Mvp07Program` を追加して expected 完全一致テストを実装
- `dotnet test --filter Mvp07` で回帰確認

## 🔍 結果
- Mvp07テストは 2件合格（失敗0）
- `VAL=15 -> RANGE=10-19`、`VAL=25 -> RANGE=OTHER` の分岐期待値を確認
- MVP07は「入力1行→出力1行」の既存パターンを維持

## 💡 学び
EVALUATEのTHRU/OTHERは最小ケースでも境界条件を固定できるため、
IF変換時の取りこぼし検出に有効。

## 🧠 思考整理
ALSOをMVP08へ分離したことで、MVP07は分岐レンジ判定に集中できた。
スコープを狭くするほど、説明可能性とテスト再現性が上がる。
