## 今回のゴール

以下を満たしてください。

1. COBOL の `READ file-name AT END ... NOT AT END ... END-READ` を最小限サポートする
2. 対象は **SEQUENTIAL入力** のみとする
3. EOF 到達時に `AT END` 側へ分岐できること
4. 読み取り成功時に `NOT AT END` 側へ分岐できること
5. 複数行入力を順次読み取れること
6. 変換結果を **自動テストで検証** できること
7. 未対応仕様は `NotSupportedException` などで明示すること

## 実装方針

### 方針1: ランタイム最小化
まず C# 側に **逐次入力用の最小ランタイム** を追加してください。

想定責務:
- テキストファイルを開く
- 1行ずつ読む
- EOF なら `false`
- 読み取り成功なら `true`
- 現在レコードを保持する

### 方針2: 変換ロジックは最小対応
`READ ... AT END ... NOT AT END` だけに絞ってコード生成してください。

変換後のイメージは概ね次の意図です。

```csharp
if (!infile.ReadNext())
{
    // AT END
}
else
{
    // NOT AT END
}
```

### 方針3: テスト先行
最低限、以下の観点を自動テスト化してください。

- 空ファイル
- 1行ファイル
- 複数行ファイル
- 入力行をそのまま出力できるケース、または行数カウント

### 方針4: 既存規約尊重
既存のディレクトリ構成、命名、テストスタイル、ランタイム配置、MVP命名規則に合わせてください。  
既存実装がある場合は **最小差分** で統合してください。  
独自流儀で大きく作り直さないでください。

### Task 2. 逐次入力ランタイムの追加
最小の sequential file reader を実装してください。

要件:
- `StreamReader` ベースでよい
- `ReadNext()` が `bool` を返す
- 現在レコードは `CurrentRecord` のようなプロパティで参照できる
- EOF時の状態は明確にする
- `IDisposable` を実装する
- 例外メッセージは追跡しやすいものにする

参考イメージ:

```csharp
public sealed class SequentialFileReader : IDisposable
{
    private readonly StreamReader _reader;

    public string CurrentRecord { get; private set; } = string.Empty;

    public SequentialFileReader(string path)
    {
        _reader = new StreamReader(path);
    }

    public bool ReadNext()
    {
        var line = _reader.ReadLine();
        if (line == null)
        {
            CurrentRecord = string.Empty;
            return false;
        }

        CurrentRecord = line;
        return true;
    }

    public void Dispose()
    {
        _reader.Dispose();
    }
}
```

※ これは概念例です。実際は既存 runtime 設計に合わせて調整してください。

### Task 3. `READ ... AT END` の変換追加
構文木または中間表現から、以下を扱えるようにしてください。

要件:

未対応構文（例: `INTO`）は、その場で `NotSupportedException` 等を投げるか、既存の未対応通知機構に乗せてください。
