## Overview

This document summarizes the incremental implementation roadmap used to validate
COBOL constructs during the development of the **COBOL → C# migration runtime**.

The strategy is:

- Implement one **COBOL feature cluster per MVP**
- Provide **minimal reproducible COBOL program**
- Provide **input / expected output test data**
- Validate using automated tests

Command used for verification:

```
dotnet test --filter MvpXX
```

Each MVP isolates a specific COBOL feature to ensure:
- deterministic behavior
- precise TODO detection
- clear conversion boundaries
